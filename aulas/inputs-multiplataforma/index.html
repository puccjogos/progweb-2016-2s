Input e movimento

Inputs multiplataforma

Exitem duas etapas principais para o desenvolvimento de sistemas de input multi-plataformas. Primeiro, é importante desacoplar as ações no mundo do jogo de um tipo de entrada. Segundo, é importante associar essa nova camada de abstração a cada uma das plataformas desejadas, adaptando conforme necessário. Para ajudar nesse processo, vamos trabalhar com a criação de objetos para auxiliar nosso input.

Através da classe Input (documentação) nós temos acesso à uma série de funcionalidades para criação de inputs multiplataforma.

A própria biblioteca também agrupa uma série de funcionalidades de input dentro do próprio Sprite, na propriedade sprite.input, do tipo InputHandler (documentação). Com ela, podemos tratar eventos como cliques, drags, checar overlaps com pixel perfect e mais.

Teclado

Exemplo: usando teclas direcionais

// dentro de create()
cursors = game.input.keyboard.createCursorKeys();
...
// dentro de update()
if (cursors.up.isDown)
{
    ...
}

Exemplo: teclas com funções

// dentro de create()
key1 = game.input.keyboard.addKey(Phaser.Keyboard.ONE);
key1.onDown.add(callback, this);

Exemplo: bloquear eventos-padrão do browser

// dentro de create()
game.input.keyboard.addKeyCapture([
    Phaser.Keyboard.LEFT,
    Phaser.Keyboard.RIGHT,
    Phaser.Keyboard.UP,
    Phaser.Keyboard.DOWN,
    Phaser.Keyboard.SPACEBAR
]);

Movimento no Phaser

Física Arcade

Para implementar movimentação no Phaser podemos alterar diretamente as coordenadas de objetos do jogo como sprites e texto. No entanto, uma forma mais consistente e poderosa de fazer esse tipo de funcionalidade passa por utilizar o sistema de física mais básico da ferramenta: Arcade Physics.

Usando o sistema de física Arcade, temos funções para controlar aceleração, rotação, cálculo de ângulos, colisões e overlaps, assim como colliders de vários tipos. Para jogos que não buscam um movimento físico realista, essa opção é mais que suficiente.

Tweens

Outra forma de movimentação possível é através de tweens (exemplos). Esta é uma alternativa interessante para jogos em que colisões são desnecessárias e que permite um alto grau de controle sobre o aspecto visual e timing dos movimentos. Ela se baseia na definição de posições e tempos, assim como diferentes formas de interpolação. É possível encadear movimentos e ativá-los de acordo com o input do usuário.
